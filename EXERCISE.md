# Coding Exercise: AI Legal Assistant Chat

## Overview

Your task is to build a simple one-page chat interface where users can interact with an AI legal assistant that has the ability to generate PDF documents.

## What We're Looking For

- **Readable and understandable code**: Clear naming, logical structure
- **Ability to utilize external APIs**: Integration with an AI service and a PDF generation API of your choice
- **Simple and smooth user experience**: Intuitive interface, proper loading states, and error handling
- **Full-stack implementation**: Good communication between the database, API, and frontend

## Prerequisites & Setup

Use the [README.md](README.md) for installation instructions and prerequisites.

## Codebase Guidance

You can review the [.cursorrules](.cursorrules) file for detailed guidance on the codebase structure, logic and how to build inside it.

## Useful Information

- No need to create authentication
- Clean the existing UI of the boilerplate and only build what is required
- You can re-use the `openai.provider.ts` for your AI Service integration - this is an option, not a requirement
- You won't need to use a lot of the boilerplate code
- You can use any tool at your disposal to help you build, but you will be asked to explain how you tackled the task and your code/decisions
